Syntax:

[MSx, Ci] = XMSEn(Sig1, Sig2, Mobj, 'Scales', 3, 'Methodx', 'coarse', 'RadNew', 0, 'Plotx', false)
  • Sig1 Time series signal, a vector (N > 10).
  • Sig2 Time series signal, a vector (N > 10).
  • Mobj Multiscale entropy object
  • Scales Number of grained time scales, a positive integer.
  • Methodx Type of graining method, one of the following strings: "coarse" / "generalized" / "modified" / "timeshift" / "imf" [See the EntropyHub Guide for more info on signal graining procedures.]
  • RadNew Radius rescaling method, an integer in the range [0 4]. When the Base entropy method specified by Mobj is XSampEn or XApEn, RadNew allows the radius threshold to be updated based on the grained signal at each time scale (). If a radius threshold value (r) is specified in Mobj, this becomes the rescaling coefficient, otherwise it is set to 0.2 (default). The value of RadNew specifies one of the following methods: 0 - no rescaling / 1 - Pooled Standard Deviation / 2 - Pooled Variance / 3 - Total Mean Absolute Deviation / 4 - Total Median Absolute Deviation
  • Plotx A plot of the multiscale entropy curve: true - Plots time scale vs entropy value / false - No plot.
  • MSx Multiscale cross-entropy estimate at each time scale, a vector of length Scales.
  • Ci Complexity index (area under the multiscale entropy curve), a scalar.

About XMSEn():

To use the multiscale cross-entropy function (XMSEn), one must first create a multiscale entropy object using MSobject() to specify the type of entropy and its parameters used in the multiscale cross-entropy estimation.
XMSEn() returns two results:
  1. MSx the cross-entropy estimate at each time scale, a vector of length Scales
  2. Ci the complexity index (area under the multiscale entropy curve), a scalar value
Note - when estimating the generalized multiscale cross-entropy (Methodx == 'generalized'), the first value of MSx will always be zero.

The 'RadNew' Argument:

Each multiscale cross-entropy function accepts a 'RadNew' name/value pair argument to perform radius distance threshold rescaling at each grained time-scale if the Cross entropy method specified by Mobj is XSampEn or XApEn.
If a radius threshold value (r) is specified in Mobj, RadNew becomes the rescaling coefficient, otherwise it is set to 0.2 (default). The value of RadNew specifies one of the following methods:
0 No rescaling
1 Pooled Standard deviation -
2 Pooled Variance -
3 Total Mean Abs Deviation -
4 Total Median Abs Deviation -

Note:

In the following examples, example signals / data are imported into MatLab using the ExampleData function from EntropyHub.
To use this function as outlined in the examples below, an internet connection is required.
ExampleData accepts any of the following strings:
  • 'uniform2' uniformly distributed random numbers in range [0 1]
  • 'gaussian2' normally distributed random numbers with mean = 0, and standard deviation = 1
  • 'randintegers2' uniformly distributed pseudorandom integers in range [1 8]
  • 'lorenz' X, Y, Z components of the Lorenz system (sigma: 10, beta: 8/3, rho: 28) [Xo = 10, Yo = 20, Zo = 10]
  • 'henon' X, Y components of the Henon attractor (alpha: 1.4, beta: .3) [Xo = 0, Yo = 0]

Example 1:

Import two signals of uniformly distributed random numbers in the range [0,1] and create a multiscale entropy object with the following parameters:
EnType = XApEn(), embedding dimension = 2, time delay = 2, radius distance threshold = 0.75
X = ExampleData('uniform2');
Mobj = MSobject('XApEn', 'm', 2, 'tau', 2, 'r', 0.4)
Mobj = struct with fields:
Func: @XApEn m: 2 tau: 2 r: 0.4000
Calculate the multiscale cross-approximate entropy over 3 coarse-grained temporal scales, where
MSx = XMSEn(X(:,1), X(:,2), Mobj)
. . . .
MSx = 13
0.4645 0.2034 0.1034
Calculate the multiscale cross-approximate entropy over 5 coarse-grained temporal scales instead.
MSx = XMSEn(X(:,1), X(:,2), Mobj, 'Scales', 5)
. . . . . .
MSx = 15
0.4645 0.2034 0.1034 0.0540 0.0270

Import two signals of normally distributed random numbers [] and create a multiscale entropy object with the following parameters:

Example 4:

Import the x and y components of the Henon system of equations and create a multiscale entropy object with the following parameters:
EnType = XFuzzEn(), embedding dimension = 4, time delay = 2, fuzzy function = 'gudermannian', fuzzy function parameter = 3, logarithm base = 10
Data = ExampleData('henon');
figure('Color', 'k')
plot(Data(:,1), Data(:,2), 'g.')
xlabel('x-component','color','g'), ylabel('y-component','color','g')
set(gca,'color','k'), axis square
Mobj = MSobject('XFuzzEn', 'm', 4, 'tau', 2, 'Fx', 'gudermannian', 'r', 3, 'Logx', 10)
Mobj = struct with fields:
Func: @XFuzzEn m: 4 tau: 2 r: 3 Fx: 'gudermannian' Logx: 10
Calculate the multiscale cross-fuzzy entropy and complexity index of the x and y-components over 4 temporal scales where the grained signal at each scale is the cumulative sum of the intrinsic mode functions from the empirical mode decomposition of the original time series signals. Return a plot of the multiscale curve with the entropy estimates also.
X = Data(:,1);
Y = Data(:,2);
 
[MSx, Ci] = XMSEn(X, Y, Mobj, 'Scales', 4, 'Methodx', 'imf', 'Plotx', true)
