Syntax:

[MSx, Ci] = MvMSEn(Data, Mobj, 'Scales', 3, 'Methodx', 'coarse', 'Plotx', false)
  • Data Multivariate dataset, NxM matrix of N (>10) observations (rows) and M (cols) univariate data sequences.
  • Mobj Multiscale entropy object
  • Scales Number of grained time scales, a positive integer.
  • Methodx Type of graining method, one of the following strings: "coarse" / "generalized" / "modified" [See the EntropyHub Guide for more info on graining methods]
  • Plotx A plot of the multiscale entropy curve: true - Plots time scale vs entropy value / false - No plot.
  • MSx Multiscale entropy estimate at each time scale, a vector of length Scales.
  • Ci Complexity index (area under the multiscale entropy curve), a scalar.

About MvMSEn():

To use the multivariate multiscale entropy function (MvMSEn), one must first create a multiscale entropy object using MSobject() to specify the type of entropy and its parameters used in the multivariate multiscale entropy estimation.
MvMSEn() returns two results:
  1. MSx the entropy estimate at each time scale, a vector of length Scales
  2. Ci the complexity index (area under the multiscale entropy curve), a scalar value
Important Note:
By default, the MvSampEn and MvFuzzEn multivariate entropy algorithms estimate entropy values using the "full" method by comparing delay vectors across all possible m+1 expansions of the embedding space as applied in [1]. These methods are not lower-bounded to 0, like most entropy algorithms, so MvMSEn may return negative entropy values if the base multivariate entropy function is MvSampEn and MvFuzzEn, even for stochastic processes...

Note:

In the following examples, signals / data are imported into MatLab using the ExampleData function from EntropyHub.
To use this function as outlined in the examples below, an internet connection is required.
ExampleData accepts any of the following strings:
  • 'uniform' uniformly distributed random numbers in range [0 1]
  • 'gaussian' normally distributed random numbers with mean = 0, and standard deviation = 1
  • 'randintegers' uniformly distributed pseudorandom integers in range [1 8]
  • 'chirp' chirp signal with the following parameters: f0 = .01, t1 = 4000, f1 = .025
  • 'lorenz' X, Y, Z components of the Lorenz system (sigma: 10, beta: 8/3, rho: 28) [Xo = 10, Yo = 20, Zo = 10]
  • 'henon' X, Y components of the Henon attractor (alpha: 1.4, beta: .3) [Xo = 0, Yo = 0]

Example 1:

Import a signal of uniformly distributed random numbers in the range [0,1] and convert it to a multivariate set of 5 sequences with 1000 samples each. Create a multiscale entropy object with the following parameters:
EnType = MvSampEn(), embedding dimension = [2,3,2,3,2], time delay = [2,2,2,2,2], radius distance threshold = 0.75
X = ExampleData('uniform');
Data = reshape(X,1000,5);
 
Mobj = MSobject('MvSampEn', 'm', [2,3,2,3,2], 'tau', 2*ones(5,1), 'r', 0.75)
Mobj = struct with fields:
Func: @MvSampEn m: [2 3 2 3 2] tau: [5×1 double] r: 0.750000000000000
Calculate the multivariate multiscale sample entropy over 3 coarse-grained temporal scales, where
MSx = MvMSEn(Data, Mobj)
. . . .
MSx = 1×3
0.062079334863415 0.005193037612927 0.000631660597576
Calculate the multivariate multiscale sample entropy over 5 coarse-grained temporal scales instead.
MSx = MvMSEn(Data, Mobj, 'Scales', 5)
. . . . . .
MSx = 1×5
0.062079334863415 0.005193037612927 0.000631660597576 0.000008467383116 0

Example 2:

Import a signal of uniformly distributed pseudorandom integers in the range [1,8] and convert it to a multivariate set of 4 sequences with 1024 samples each. Create a multiscale entropy object with the following parameters:
EnType = MvCoSiEn(), embedding dimension = [3,3,3,3], angular threshold = .4, normalization of data to zero mean = true.
X = ExampleData('randintegers');
Data = reshape(X,1024,4);
Mobj = MSobject('MvCoSiEn', 'm', 3*ones(4,1), 'r', 0.4, 'Norm', 2)
Mobj = struct with fields:
Func: @MvCoSiEn m: [4×1 double] r: 0.400000000000000 Norm: 2
Calculate the multivariate multiscale cosine similarity entropy over 5 temporal scales using the 'modified' graining procedure where
MSx = MvMSEn(Data, Mobj, 'Scales', 5, 'Methodx', 'modified')
. . . . . .
MSx = 1×5
0.608564637395733 0.696958694945509 0.756035969530514 0.784935740470017 0.804721512742983

Example 3:

Import the x and y components of the Henon system of equations and create a multiscale entropy object with the following parameters:
EnType = MvPermEn(), permutation entropy variant = 'phase', lograithm base = 2
Data = ExampleData('henon');
figure('Color', 'k')
plot(Data(:,1), Data(:,2), 'g.')
xlabel('x-component','color','g'), ylabel('y-component','color','g')
set(gca,'color','k'), axis square
Mobj = MSobject('MvPermEn', 'Typex', 'phase', 'Logx', 2)
Mobj = struct with fields:
Func: @MvPermEn Typex: 'phase' Logx: 2

[MSx, Ci] = MvMSEn(Data, Mobj, 'Scales', 7, 'Methodx', 'generalized')
