Syntax:

[MFuzz, B0, Bt, B1] = MvFuzzEn(Data, 'm', 2*ones(size(Data,2),1), 'tau', ones(size(Data,2),1), 'r', 0.2, 'Norm', false, 'Logx', exp(1))
  • Data Multivariate dataset, NxM matrix of N (rows, >10) observations and M (cols) univariate data sequences
  • m Embedding dimension, a vector of M positive integers.
  • tau Time delay, a vector of M positive integers.
  • Fx Type of fuzzy function for distance transformation, one of the following strings: 'default' / 'sigmoid' / 'modsampen' / 'gudermannian' / 'triangular' / 'trapezoidal1' / 'trapezoidal2' / 'z_shaped' / 'bell' / 'gaussian' / 'constgaussian'. See the EntropyHub Guide (.pdf) for more info.
  • r Parameters of the fuzzy function specified by Fx, a 1 element scalar or a 2 element tuple of positive values depending on the fuzzy function as shown above. Default/Sigmoid/ModSampEn/Trapezoidal2/Bell: Two element vector; Gudermannian/Triangular/Trapezoidal1/Gaussian/Constgaussian/Z-Shaped: A scalar value
  • Norm Option to normalize data to unit variance, true or false.
  • Logx Logarithm base in Shannon's entropy formula, a positive scalar.
  • MFuzz Multivariate fuzzy entropy estimate
  • B0 Average delay vector distance for given m embedding dimensions.
  • Bt Average delay vector distance for the joint total m+1 embedded subspace.
  • B1 Average delay vector distance for each possible m+1 embedded subspaces.

About MvFuzzEn:
The entropy value returned as MFuzz is estimated using the "full" method [i.e. -log(Bt/B0)] which compares delay vectors across all possible m+1 expansions of the embedding space as applied in [1][3]. Contrary to conventional definitions of fuzzy entropy, this method does not provide a lower bound of 0!! Thus, it is possible to obtain negative entropy values for multivariate fuzzy entropy, even for stochastic processes...
Alternatively, one can calculate MFuzz via the "naive" method, which ensures a lower bound of 0, by using the average vector distances for an individual m+1 subspace (B1) [e.g. -log(B1(1)/B0)], or the average for all m+1 subspaces [i.e. -log(mean(B1)/B0)]. To maximize the number of points in the embedding process, this algorithm uses N-max(m*tau) delay vectors and not N-max(m)*max(tau) as employed in [1][3].

Note:

In the following examples, signals / data are imported into MatLab using the ExampleData function from EntropyHub.
The ExampleData function requires an internet connection to download and import the required datasets!
ExampleData imports datasets with any of the following strings:
  • 'uniform' {[N,1] vector} uniformly distributed random numbers in range [0 1]
  • 'gaussian' {[N,1] vector} normally distributed random numbers with mean = 0, and standard deviation = 1
  • 'randintegers' {[N,1] vector} uniformly distributed pseudorandom integers in range [1 8]
  • 'chirp' {[N,1] vector} chirp signal with the following parameters: f0 = .01, t1 = 4000, f1 = .025
  • 'lorenz' {[N,3] matrix} X, Y, Z components of the Lorenz system (sigma: 10, beta: 8/3, rho: 28) [Xo = 10, Yo = 20, Zo = 10]
  • 'henon' {[N,2] matrix} X, Y components of the Henon attractor (alpha: 1.4, beta: .3) [Xo = 0, Yo = 0]

Example 1:

Import a vector of uniformly distributed random numbers in range [0 1] and convert it to a multivariate set of 5 sequences with 1000 samples each. Calculate the multivariate fuzzy entropy using the default parameters (embedding dimensions (m) = [2,2,2,2,2], time delay (tau) = [1,1,1,1,1], r = [0.2,2], fuzzy membership function = "default").
X = ExampleData('uniform');
Data = reshape(X,1000,5);
MFuzz = MvFuzzEn(Data)
MFuzz =
0.110827810371706

Example 2:

Import a vector of uniformly distributed random numbers in range [0 1] and convert it to a multivariate set of 4 sequences with 1250 samples each. Calculate the multivariate fuzzy entropy for embedding dimensions (m) = [1,2,3,4], time delay (tau) = [1,1,1,1,1], r = [0.2, 2] and a "default" fuzzy membership function.
X = ExampleData('uniform');
Data = reshape(X,1250,4);
MFuzz = MvFuzzEn(Data, 'm', [1,2,3,4])
MFuzz =
0.110863681396715
Repeat the previous calculation, but change the time delay values to [4,3,2,1] and change the fuzzy membership function parameters to r = [0.5, 1.5].
MFuzz = MvFuzzEn(Data, 'm', [1,2,3,4], 'tau', [4,3,2,1], 'r', [0.5,1.5])
MFuzz =
0.042339025934306

Example 3:

Import a vector of 4096 uniformly distributed random integers in range [1 8] and convert it to a multivariate set of 8 sequences with 512 samples each.
Calculate the multivariate fuzzy entropy for embedding dimensions (m) = [1,1,1,1,2,2,2,2], a "trapezoidal1" fuzzy function, and fuzzy function parameter r = 4.
Return the number of matched vectors for the m-dimensional embedding space (B0), the joint total m+1 dimensional embedding space (Bt), and each individual m+1 dimensional embedding space (B1).
X = ExampleData('randintegers');
Data = reshape(X,512,8);
[MFuzz, B0, Bt, B1] = MvFuzzEn(Data, 'm', [1,1,1,1,2,2,2,2], 'Fx', 'trapezoidal1', 'r', 4)

Example 5:

Import a sequence of 5000 normally-distributed random numbers and reshape them into a multivariate dataset of 5 sequences of 1000 samples each. Calculate the multivariate fuzzy entropy using a distance threshold of r=1.2 and a "gaussian" fuzzy function (and all other default parameters) for each of the five possible m+1 embedding subspaces.
X = ExampleData('gaussian');
Data = reshape(X,1000,5);
[~, B0, ~, B1] = MvFuzzEn(Data, 'Fx', 'gaussian' , 'r', 1.2);
MFuzz = -log(B1/B0)'
MFuzz = 5×1
0.101745522288216 0.097136069792768 0.112329375567604 0.105442419625667 0.105357408333410
